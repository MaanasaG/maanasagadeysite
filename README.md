# maanasagadey.org

Personal site for **Maanasa Gadey** — Software Engineering Manager at Workday (Workday Student platform).

The site is a single, scroll-driven page. As the visitor scrolls from the hero down to Contact, a plant grows alongside the text — a 121-frame WebP sequence scrubbed against scroll progress. The plant's background is alpha-keyed to transparent at build time, so the page background flows through and the animation reads like a watermark drawn directly on the page.

---

## Stack

- **Next.js 15** (App Router, `output: 'export'` — fully static)
- **React 19** + **TypeScript**
- **Tailwind CSS 3**
- **MDX** for blog posts (`@next/mdx`)
- **Source Serif 4** + **Inter** + **JetBrains Mono**, self-hosted via `next/font`

No UI library, no animation library, no analytics, no cookie banner.

## Deployment — Cloudflare Pages

The simplest path:

1. **Push the repo to GitHub** (already done if you're reading this).
2. **Cloudflare Pages → Create a project → Connect to Git** → select this repo.
3. Build configuration:
   - **Framework preset:** Next.js (Static HTML Export)
   - **Build command:** `npm run build`
   - **Build output directory:** `out`
   - **Node version:** 22 (set in env: `NODE_VERSION=22`)
4. **Custom domain:** add `maanasagadey.org` and set the apex DNS record per Cloudflare's instructions.

That's it. No serverless functions, no edge runtime — pure static.

## The plant animation

### Source

`plant_growth.mp4` at the repo root — a 5-second clip of a plant growing on a near-white background (sampled `#F3F4F2`). 1924×1076, 24 fps, 121 frames total — already at the target ~120 frames, no resampling needed.

### Frame pipeline

The frames in `public/frames/` are generated by `scripts/extract-frames.sh`. Two stages:

```bash
# Stage 1 — ffmpeg: scale to 1200px wide, alpha-key the near-white background
ffmpeg -y -i plant_growth.mp4 \
  -vf "scale=1200:-2:flags=lanczos,format=rgba,colorkey=color=0xF3F4F2:similarity=0.10:blend=0.06" \
  -fps_mode passthrough \
  /tmp/frame_%04d.png

# Stage 2 — cwebp: PNG → WebP at q=80, alpha preserved
for f in /tmp/frame_*.png; do
  cwebp -q 80 -alpha_q 100 -mt -quiet "$f" -o "public/frames/$(basename "${f%.png}.webp")"
done
```

Both stages are wrapped in `npm run frames`. Total payload is **~2.0 MB for all 121 frames**, well under the 3 MB performance budget.

Requires `ffmpeg` and `cwebp`:

```bash
brew install ffmpeg webp
```

### Render

`components/PlantCanvas.tsx`:

- Preloads all 121 frames as `HTMLImageElement`s with `img.decode()`.
- Renders the current frame to a `<canvas>` via `drawImage` (object-fit: cover).
- A `scroll` listener throttled with `requestAnimationFrame` maps `window.scrollY` → frame index 0…120.
- On viewports `<768px` or with `prefers-reduced-motion: reduce`, swaps to a single static frame (frame 60 — mid-growth) and skips the scroll binding.

Because the WebP frames have transparent backgrounds, the canvas is just `clearRect` + `drawImage` — no CSS `mix-blend-mode`, no in-canvas multiply tricks, no visible boundary between the canvas and the page.

## Adding a blog post

1. Drop a new `.mdx` file in `content/posts/`. Use plain Markdown for the body; the styles in `app/globals.css` under `.essay` handle typography.
2. Register it in `lib/posts.tsx`:

   ```ts
   import NewPost from '@/content/posts/new-post.mdx'

   export const posts: Post[] = [
     // …
     {
       slug: 'new-post',
       title: 'New post title',
       dek: 'A one-sentence dek.',
       date: '2026-06-01',
       dateLabel: 'June 2026',
       Component: NewPost,
     },
   ]
   ```

The Writing section renders posts in the order of the `posts` array — newest first by convention.

## Design tokens

| Token        | Value     | Where                                        |
| ------------ | --------- | -------------------------------------------- |
| `--page`     | `#F2F4EF` | Page background — slightly green-tinted off-white |
| `--ink`      | `#1E2D1C` | Body text — deep forest                      |
| `--muted`    | `#5E6E5C` | Secondary text                               |
| `--sage`     | `#6E8A6E` | Accent — section labels, hairlines           |
| `--sage-dark`| `#3F5A3F` | Hover states, numbered list markers          |

Body 17 px / 1.7 line-height (Inter). Headings in **Source Serif 4 regular** — never bold. Single-column layout, max content width 40 rem (640 px), centered with the plant occupying the right ~65% of the viewport on desktop.

## Accessibility

- Page background contrast: WCAG AA against the body ink color
- The plant canvas is `aria-hidden="true"` — no semantic content
- Focus rings: 1px solid sage at 2px offset
- `prefers-reduced-motion`: scroll-tied animation is skipped, static frame 60 is shown instead
- Semantic HTML: `<main>`, `<section>`, `<article>` for posts
- No analytics, no tracking pixels, no cookie banner
